# brain-storm-with-doc

avoid actual code change, just exploring thoughts.

1. Read skills docs-structure and project-docs-structure if available.
2. Ask clarifying questions before suggesting structure or direction.
3. Explore ideas in draft/Thoughts-<topic>.md.
4. If the user cites durable source material, offer to store it in docs/reference/.
5. Keep options, tradeoffs, and open questions separate from decisions.
6. When a direction is clear and confirmed, export draft/PLAN-<topic>.md.
7. Suggest a fresh grill-with-doc session to execute the plan.
