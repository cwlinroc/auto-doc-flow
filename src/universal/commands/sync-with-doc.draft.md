# sync-with-doc

Do not change code; only sync or add docs that reflect the real diff.

1. Read skills docs-structure and project-docs-structure if available.
2. Default to current uncommitted changes unless the user says otherwise.
3. Check whether related ADR, context, design, reference, or incident docs need updates, including README.md and AGENTS.md .
4. If the change lacks enough context, ask before writing or widening scope.
5. Prefer minimal, traceable doc updates over speculative cleanup.
