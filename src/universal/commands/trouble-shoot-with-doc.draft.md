# trouble-shoot-with-doc

1. Read skills docs-structure and project-docs-structure if available.
2. Start in draft/Problem-<topic>.md and separate observations from conclusions.
3. Ask enough questions to classify the issue.
4. If useful source material appears, offer to store it in docs/reference/.
5. When a conclusion is confirmed, record it in the docs/incident/ and append immutable follow-up notes.
    - write down when does this happened too, while-dev, while-test, or while-production.
6. If the fix needs code changes, export draft/PLAN-<topic>.md instead of coding here, and suggest grill-with-doc for the implementation session.
