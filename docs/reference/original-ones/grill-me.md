# Grill Me

Interview the user relentlessly about every aspect of their plan or design until reaching a shared understanding. Walk down each branch of the design tree, resolving dependencies between decisions one-by-one.

## Step 1: Initialize the interview

Begin the session by stating the objective: to stress-test their plan or design, walking down each branch of the decision tree, and resolving dependencies between decisions one-by-one.

## Step 2: Explore the codebase first

If there are questions that can be answered by exploring the codebase (e.g., existing structure, dependencies, naming conventions, or configuration patterns), explore the codebase first instead of asking the user. Use the allowed search and read tools to gather context.

## Step 3: Ask questions relentlessly

Ask clarifying or design questions to stress-test the plan:
- Propose your own recommended answer/approach for each question you ask.
- **Ask the questions one at a time**, waiting for the user's feedback before moving to the next question.
- Walk down each branch of the decision tree systematically.
